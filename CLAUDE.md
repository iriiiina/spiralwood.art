# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Static one-page site for the musician **Spiral Wood**, served from GitHub Pages at https://spiralwood.art (CNAME points the apex domain at the `iriiiina/spiralwood.art` repo). Vanilla HTML + CSS, **no build step, no JS framework, no package manager** — files in the repo root are what gets served. Open `index.html` directly in a browser to preview.

## Architecture

Everything renders from three files: `index.html`, `style.css`, `manifest.json`. The single page is structured as discrete sections in `index.html`, each styled by a matching block in `style.css` (the CSS is sectioned with `/* HEADER */`, `/* SOCIAL */`, `/* LIVE */`, etc. — keep that 1:1 mapping when adding things).

Page sections, in order:
- `#header` — hero with the SVG `#logo`. The inline `<script>` in `<head>` rotates `#rotate` and fades `#header` proportional to `documentElement.scrollTop`. This is the only JS on the site.
- `#social` — outbound platform links. Icons are inline SVGs sourced from [simpleicons.org](https://simpleicons.org/).
- `#press-kit` — link to `epk-spiral-wood.pdf` at repo root (also listed in `sitemap.xml`).
- `#live` — concert history, grouped into `.live-row` per year, each entry a `.live` block with a `<picture>` (webp + png) and `.live-description`. Marked up with `schema.org/MusicEvent` microdata.
- `#albums` — Spotify embed iframe.
- `#pins`, `#gallery` — decorative.

SEO/structured data is everywhere: every clickable thing carries `itemscope`/`itemtype`/`itemprop` attributes. Preserve them when editing — the site is validated against [validator.schema.org](https://validator.schema.org/) and [konversioon.ee](https://konversioon.ee/).

## Common edits

**Adding a live show:** edit `index.html`, find the `.live-row` for the correct year (or add a new `.live-row` block at the top of `#live` if it's a new year), and append a `.live` block. Copy the structure of an existing entry — `<picture>` with both `.webp` and `.png` `<source>` tags, then the `.live-description` with `schema.org/MusicEvent` `itemprop`s (`name`, `startDate`, `location`).

**Adding an image:** always produce **both** WebP and PNG versions and reference them via `<picture><source srcset=".webp"><source srcset=".png"><img src=".png"></picture>`. The hero forest image also has 480/640/768/1024/2048/4096 width variants in `img/` — keep that pattern if replacing it. Resize/reformat with [squoosh.app](https://squoosh.app/).

**Favicons:** the `favicon/` directory is generated by [realfavicongenerator.net](https://realfavicongenerator.net); regenerate the full set rather than hand-editing individual sizes.

**New page or URL:** update `sitemap.xml` (currently lists `/` and `/epk-spiral-wood.pdf`).

## Deploy

Push to `main` — GitHub Pages serves the repo root directly. No CI, no build.
